# Spec: About

## Purpose

The About section presents the owner's bio text, skill progress bars, an about photo with an experience badge, and a "See More" button that opens a modal with four sub-sections: Personal Info, Achievements, Experience, and Education. Interactive modal behavior is implemented as a React island.

---

## Requirements

### Requirement: About Section Structure

The system MUST render an About section with `id="about"` containing: a left column with bio text and skill progress bars, and a right column with the about photo and a `"6+ Years of Experience"` badge overlay. A "See More" button MUST be present below or alongside the about content.

#### Scenario: About section layout is correct

- GIVEN the page is loaded
- WHEN the `#about` section is inspected
- THEN a bio text block, skill progress bars, about photo, experience badge, and "See More" button MUST all be present

---

### Requirement: Skill Progress Bars

The system MUST render animated progress bars for each skill defined in the skills content source. Each bar MUST display the skill name and percentage value. The exact skills and percentages are: React.js 95%, React Native 90%, Express 90%, Next.js 90%, TypeScript 85%, PostgreSQL 80%.

#### Scenario: All six skills are rendered

- GIVEN the about section is rendered
- WHEN the skill bars are inspected
- THEN exactly six skill bars MUST be present: React.js, React Native, Express, Next.js, TypeScript, PostgreSQL

#### Scenario: Skill percentages match specification

- GIVEN the about section is rendered
- WHEN each skill bar's width or aria value is inspected
- THEN React.js MUST be 95%, React Native 90%, Express 90%, Next.js 90%, TypeScript 85%, PostgreSQL 80%

#### Scenario: Skill data comes from content collection, not hardcoded HTML

- GIVEN the skills content collection contains the six skill entries
- WHEN the About component renders
- THEN the skill bars MUST be generated by iterating over the collection data

---

### Requirement: About Photo with Experience Badge

The system MUST render the about photo using `astro:assets` image processing with a non-empty `alt` attribute. An overlay badge reading `"6+ Years of Experience"` MUST be visually positioned over or adjacent to the photo.

#### Scenario: About photo renders correctly

- GIVEN the about section is rendered
- WHEN the about photo is inspected
- THEN it MUST have a non-empty `alt` attribute and explicit dimensions to prevent CLS

#### Scenario: Experience badge is visible

- GIVEN the about section is rendered
- WHEN the badge element is inspected
- THEN it MUST contain the text `"6+ Years of Experience"`

---

### Requirement: "See More" Modal — Opening

The system MUST render a React island (`client:visible`) that manages a modal opened by the "See More" button. The modal MUST use `react-modal` with `className="custom-modal about-popup-wrapper"`, `overlayClassName="custom-overlay"`, and `closeTimeoutMS=500`.

#### Scenario: "See More" button opens the modal

- GIVEN the about section is rendered and the modal is closed
- WHEN the user clicks the "See More" button
- THEN the modal MUST open and its content MUST be visible

#### Scenario: Modal has correct react-modal class names

- GIVEN the modal is open
- WHEN the modal DOM element is inspected
- THEN it MUST have class `custom-modal` and `about-popup-wrapper`
- AND the overlay MUST have class `custom-overlay`

---

### Requirement: "See More" Modal — Content Sub-sections

The modal MUST display four sub-sections accessible via tabs or a structured layout: Personal Info, Achievements, Experience, and Education. All content within these sub-sections MUST come from content collections or structured data, not hardcoded template strings.

#### Scenario: All four sub-sections are present in the modal

- GIVEN the modal is open
- WHEN the modal content is inspected
- THEN Personal Info, Achievements, Experience, and Education sub-sections MUST all be accessible

### Requirement: Modal Close Behavior

The modal MUST close when the user clicks the close button (which uses `/img/svg/cancel.svg` icon). The modal MUST also close when the user clicks the overlay backdrop. The close animation MUST respect the `closeTimeoutMS=500` setting.

#### Scenario: Close button dismisses the modal

- GIVEN the about modal is open
- WHEN the user clicks the close button
- THEN the modal MUST begin closing and be fully hidden after 500ms

#### Scenario: Overlay click dismisses the modal

- GIVEN the about modal is open
- WHEN the user clicks outside the modal content (on the overlay)
- THEN the modal MUST close

---

### Requirement: About Section AOS Animations

The about section MUST use AOS attributes: `data-aos="fade-right"` on the left column (bio + skills), `data-aos="fade-left"` on the right column (photo), with `data-aos-duration="1200"` and `data-aos-delay` values in 100ms steps for staggered children.

#### Scenario: AOS attributes are applied correctly

- GIVEN the about section is rendered
- WHEN the animated elements are inspected
- THEN the left column MUST have `data-aos="fade-right"` and the right column MUST have `data-aos="fade-left"`
